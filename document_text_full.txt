Abstract

 This document describes VisionMate, a smartphone-centred assistive system for people with visual impairments that combines on-device obstacle perception, offline face recognition, turn-by-turn voice navigation, and guardian safety synchronization in a single mobile application. The system implements a dual-zone safety model: a frontal zone for near-term collision detection and a lower path zone that estimates walkability and announces transitions between blocked and clear states. Edge inference is performed locally using YOLOv8n-int8 for object detection and MobileFaceNet for face embedding extraction (TensorFlow Lite), minimising cloud dependence for privacy-sensitive perception tasks. Interaction is audio-first, using text-to-speech for continuous situational feedback and speech input for destination entry; hardware volume-button triggers and inertial fall-detection enable quick user actions. A guardian mode shares realtime location and device status via Firebase (authentication and sync), while a local SQLite cache supports offline face matching. The implementation demonstrates that commodity smartphones can deliver an integrated, low-cost assistive solution that improves autonomy and safety for visually impaired users.

Introduction

Visual impairment can severely limit an individual’s independence, safety, and confidence during routine activities. Tasks that sighted people take for granted—avoiding obstacles, recognising known people, and following a safe path—become difficult to perform unaided. Traditional aids such as the white cane help with immediate physical barriers but do not provide semantic scene information or person recognition. Modern assistive devices that offer richer functionality are often costly or require specialised hardware.

VisionMate is a mobile application that transforms an ordinary smartphone into an assistive companion for people with visual impairments. The app provides real-time object detection and tracking, offline face recognition for identifying registered contacts, voice-guided navigation, and a guardian mode for live location sharing. Core feedback is delivered via audio so the interface remains accessible without visual dependence. By combining on-device perception, offline caching, and optional cloud sync, VisionMate aims to deliver an affordable, holistic solution that enhances mobility, social interaction, and personal safety.

Motivations

This project is motivated by the need for an accessible, reliable, and practical assistive system that increases independence and safety for people with visual impairments. Many existing solutions focus on a single capability (for example, navigation or object recognition), often require continuous internet connectivity, or depend on costly specialized hardware. VisionMate addresses these gaps by providing multiple assistive functions within a single smartphone app, delivering realtime support while prioritising usability, privacy, and costeffectiveness. By combining ondevice deep learning, voice interaction, and optional cloud services, the project aims to empower users with greater autonomy, confidence, and dignity.

Project Overview

VisionMate transforms a commodity smartphone into an integrated assistive platform that bundles realtime object detection, offline face recognition, voiceguided navigation, and guardian safety features. The system emphasises ondevice inference (TensorFlow Lite models) and local caching to reduce cloud dependency while using Firebase services selectively for authentication, realtime location sharing, and guardian sync. The app’s audiofirst interaction model and minimal hardware requirements make it practical and affordable for broad deployment. Overall, the project targets a holistic, usercentric solution that balances functionality, responsiveness, and privacy.

Problem Statement

Visually impaired people face continuous challenges when navigating everyday environments: avoiding obstacles, identifying familiar people, and following safe routes. Traditional aids such as the white cane provide important tactile feedback but cannot convey semantic or contextual scene information. Commercial “smart” assistive devices can be expensive, bulky, or restricted in capability, limiting accessibility for many users. There is therefore an urgent need for an affordable, widely available assistive technology that provides rich, realtime perception and guidance without requiring specialised hardware. This project aims to meet that need by leveraging existing smartphones and intelligent software to improve mobility, situational awareness, and personal safety.

Objectives

Upon completion, this project anticipates to meet the following results:

Implement ondevice, realtime object detection (e.g., car, chair, stairs) using the phone camera.

Provide offline face recognition for registered contacts via ondevice embeddings and local caching.

Develop voiceactivated navigation with bookmarking and speechinitiated route start.

Implement a secure guardian mode for remote realtime location and devicestatus monitoring.

Design an audiofirst, accessible UI that works with screen readers and supports control via physical keys and English voice commands.

Domain analysis identifies the users, environment, stakeholders and external systems that shape VisionMate’s requirements. It clarifies what the system must do, what data it will handle, and constraints affecting performance and usability so the final solution is practical, secure and usercentred.

Literature review

Zhao et al. [1] proposed Accessibility Bot, a research prototype for assisting individuals with visual impairments (VIPs) to identify their friends and perceive facial information in the process of social activities. Integrated into Facebook Messenger, the system uses the phone's camera to identify faces in real time, and takes advantage of cutting-edge computer vision models that have been trained by using existing tagged photos on Facebook, that eliminates the need for manually creating training datasets. The application extracts important information about the faces, such as which face, location of the face in relation to the others, expressions and attributes like facial hair, that is fed back to the user via the audio feedback via a screen reader. While the underlying recognition algorithm was able to achieve greater than 97% accuracy on the PIPA dataset, in a study involving six VIPs over a period of seven days, it was noted that in practice "perceived accuracy" was often less because of issues with aiming the camera and inconsistent photo quality. Ultimately, the research was able to offer some deep insights into the practical problems of implementing assistive face recognition technology in uncontrolled environments outside of the lab.

Kamran et al. [2] introduced "Visually," which is an AI-based mobile application that is intended to make visually impaired people more autonomous thanks to real-time multi-modal recognition. The system uses the (YOLOv5) architecture for object detection for efficiency and uses Google's ML Kit for integrated Text-to-Speech audio feedback. For locations where connectivity is limited, deep learning models are run through TensorFlow Lite, so that they can be run offline. The application was trained with a diverse and augmented data of essential day-to-day objects so as to be more robust in different real world situations. Experimental results demonstrated the high degree of identification precision with the accuracy rate of 99 percent and 98 percent for person and vehicle recognition, respectively.

Zhang et al. [3] proposed the NaviGPT, which is a high fidelity real-time Artificial Intelligence (AI) aided mobile navigation system, which aims to improve the traveling experience and enhance the safety of people with visual impairments (PVI). Using the LiDAR technology, vibration feedback integrated with large language model (GPT-4) response, the system integrates the obstacle detection into a single interface to help with navigation in a continuous manner. It uses a mobile camera for taking a picture of the environment and Apple maps for the location information, so the (LLM) can then create guidance on the context, an actual description of the scene and safety alerts. To make sure that the immediate awareness of surroundings are ensured, the prototype has dynamic vibration frequencies which are increased as the objects are getting closer, effectively simulating a digital white cane. As opposed to existing tools, NaviGPT gives brief feedback of an average of around 28 words per each response to ensure fast delivery of information in dynamic environments without the need for app-switching.

Kuriakose et al. [4] investigated the user experience of DeepNAVI, an artificial intelligence (AI)-based smartphone navigation assistant for independent mobility enhancement of people living with vision impairments. The system consists of a mobile phone camera and captures video in real-time, which is processed by some of the embedded deep learning algorithms based on small microprocessors to detect types and distance of obstacles and positions without any internet connectivity. Navigation information is provided to the user through bone-conduction headphones and the system can be operated with the assistance of voice commands, in order to ensure a hands-free experience. A qualitative study of 13 users in both indoor and outdoor environments found that while users liked the portability and ability of the system to sense hindrances in their path at real-time, most users preferred to use the system as a tool in addition to a traditional white cane, rather than as a stand-alone tool. Although some of the participants expressed concerns about taking the technology especially with its reliability and trust, it was concluded that the assistant was successful in improving the awareness about the environment and also brought a sense of independence.

Albarico et al. [5] proposed a mobile based vision assistants application, which was designed to be an affordable "one-stop-shop" solution for visually impaired and illiterate people and was specifically aimed at people in the rural areas or in less economically advanced areas. Rooted in the Model Human Processor (MHP) theory and Human Computer Interaction principles, the system includes speech-to-text (STT), text-to-speech (TTS) and YOLOv3-based object detection for enabling real-time interaction with the environment and object recognition. One of the unique features of the application is the SOS and Emergency Communication module which enables the user to quickly message or call designated guardians at the tap of a button. Prototype testing in the Philippines showed significant levels of user satisfaction and reliable functionality with emergency alerts being delivered successfully in less than 10 seconds even in areas with poor network coverage. While the STT feature went out with about 80% accuracy, there are a number of areas in the study that still need to be improved that include sensitivity to noise present in the background, difficulty with speech in mixed Filipino-English, and perceived "robotic" quality of the voice output, according to the study. Ultimately, the system is yet a big step towards inclusive technology with the assistive conversion tools combined with safety solutions to encourage a higher level of user autonomy and confidence.

Chen et al. [6] proposed BlindNavi which is a mobile-based application used in assisting the visually impaired person to navigate the external world and do it safely and independently. The system uses micro-location technology through iBeacons that can give precise and short-range directions that is more optimistic in walking navigation as compared to conventional GPS. It translates visual navigation information into cues of more than one sense such as ringing of doorbell or smell of a bakery to help the users to check their whereabouts. The guidance is presented by means of a flat user interface, as well as, multimodal feedback, e.g. a set of vibration patterns and voice messages to turn instructions on. The app does not rely on any abstract measurements such as meters but instead relies on the use of clock positions and voice commands with first action in order to correspond with the conventional Orientation and Mobility (O&M) training. The blind on-road tests proved that these features made the navigation process really easier and decreased the confusion of the users. Finally, the app helps to build a reliable mental map about the surrounding, which makes the users more independent and autonomous while traveling.

Kumar et al. [7] proposed an AI Assistant for Visually Impaired, a smart assistive system with advanced functionality to increase the autonomy and mobility by perceiving the real-time environment. The solution uses a combination of computer vision and machine learning running on a light weight wearable device using OpenCV for video processing, as well as Keras-based InceptionV3 and LSTM architecture for advanced captioning of images. A defining feature of the system is its complete voice command interface which allows the user to switch on facial recognition, OCR to read signs and descriptions of scenes by simply commanding the system with words such as "recognize" or "read". The system has amazing performance, that is, 85.2% in image recognition accuracy, and the position tracking accuracy of 82.6%, while still ensuring the near-instantference time of 250 milliseconds. Results suggest the system is an effective way of bridging the accessibility gap by providing intuitive audio feedback through the implementation of Google Text-To-Speech although there is still room for improvement in naturalness of speech (currently a 3.5 MOS), and text coherence which are areas that could be optimized in the future. This work provides a basic framework for the future trends in intelligent infrastructure integration and more effective object detection.

Pydala et al. [8] proposed SMART_EYE, an efficient navigability and obstacles detection system to facilitate visually-impaired persons in unknown environments. The system consists in the use of smart application combined with Artificial Intelligence and sensor technology, through the ultrasonic sensors for the detection of obstacle and through the Raspberry pi camera for the realization in real time of the image classification. By the use of computer vision and new approach of the measuring of the object's proximity, SMART_EYE offers not only situational awareness but also voice operated instructions that enables to detect not only several objects, but longer objects such as doors, walls, etc. together if they are used simultaneously. The hardware prototype comprises of Raspberry Pi 3, Bluetooth connectivity and Android based interface "Speak Data" that can convert the visual and sensor data into audible data. Testing result revealed good accuracy during the daytime, the accuracy rate could reach up to 95% for human detection and 91% for laptop, however the accuracy rate was considerably reduced in low-light condition at night. Ultimately, the system is intended to be a cost-effective and portable alternative to traditional bulky assistive devices and is aimed to bring the independent mobility and the overall quality of life to the blind community.

Khan et al. [9] proposed an Intelligent Assistive Device, a robust system which made use of Artificial Intelligence (AI), Computer Vision and the Internet of Things (IoT) to facilitate enhanced independence of visually impaired people. Understanding this, with numerous research efforts being theoretical and not real implementation, this project is towards wearable, Raspberry Pi 4. The architecture which is used in the system is the YOLO-FastestV2 which is optimized to offer real time performance of 18.8 FPS on edge devices with good accuracy for the safety critical objects. Key functionalities include: multi-modal feedback for independent navigation, the ability to detect obstacles and a hybrid distance measurement strategy through computer vision coupled with ultrasonic sensors to ensure precision within 0.2 to 2% margin of error. Beyond navigation, the device incorporates other important secondary functions like face and currency recognition, wet floor alert, IoT based guardian's dashboard with live location and SOS dial out for emergency etc. Performance results showed the outstanding proficiency, that is, a detection rate reaching to 99% detection success for pedestrians and 98% for vehicles and hence it is robust and cost effective alternative to bulky and expensive traditional aids.

Okolo et al. [10]  offered a detailed review of the State-of-the-Art navigation assistance for the visually impaired persons, taking into consideration the integration of the Artificial Intelligence (AI), Internet of Things (IoT) and mobile technologies. The article divides assistive systems into three major modules, navigation (wayfinding), object detection (hazard avoidance), and the human-machine interface. The authors go through many different sources of sensing, and highlight that although depth cameras such as LiDAR can provide rich spatial information, they are often bulky and hard to operate in brightly-lit environments. A key result of the research is the need for multi-modal feedback with a combination of audio, tactile and haptic signals, so that the system will work well in different situations e.g. with tactile alerts in a noisy urban environment where audio feedback to the user is masked. The review highlights that deep learning architectures such as (YOLO and SSD) have improved the object recognition performance at real time significantly but many systems are limited in the processing latency and adaptability to the environmental conditions. Ultimately, the study emphasizes that the focus of these technologies to be adopted on a large-scale needs to be on portability, affordability, and low learning curve for the individual utilizing the technology.

Sun et al. [11] proposed a dual mode and multilingual mobile application with InnoSpire Glasses to help visually impaired and blind (VIB) to navigate in a public place and recognize objects. The system has a seamless switching mechanism in order to switch between mode online and mode offline using a prompt-engineered model of GPT-4o and YOLOv5 for object detection and Phi-3-mini small language model. To make the application more accessible it has speech to text and text-to-speech modules that enable users to communicate using natural language in English, Spanish, Mandarin or Cantonese. Experimental evaluations have proven that the system is very efficient with the mean image-description latencies of 4.6 seconds on-line and 1.5 seconds off-line. Furthermore, the system had a very high average precision of 99% with speech recognition in different noisy, real-world environments. User studies including VIB as well as sighted users proved the efficacy of the prototype with the majority rating navigation and object identification capability as "Good" or better.

Brilli et al. [12] introduce AIris, an A.I.-based wearable assistive technology for visually impaired people that assist them using the recognition (in real-time) and audio feedback about the surrounding. The system utilizes a camera mounted in 3D printed smart glasses to capture visual data which is processed onsite and through the cloud via deep learning models for face recognition, object detection, scene description, optical character recognition and currency identification. The information that is interpreted is translated to speech and sent to the user via the attached earphones to allow safe navigation and interaction with the indoor and outdoor environments. Prototype testing showed to have high performance in recognition and practical usability, however, improvements are still required in order to achieve high processing latency and user comfort during long term continuous wear.

Kianpisheh et al. [13] proposed a wearable face recognition system, which is aimed to help the visually impaired people (VIPs) recognize "weak-tie" relationships such as the causal acquaintances. The system uses a mobile phone worn around the neck to opportunistically capture and store undistorted face images and the contextual data during social interactions without the user intervention. This prototype uses the detection of the presence of faces within a certain distance and head pose range, which is used to identify the social engagement and filter out high-quality images for purposes of populating a database. To support recognition, the system has entry association with the time, location and duration of the interaction along with the ability to record custom audio description. In tests of technology probes, people showed that they were able to use these accumulated contextual cues and audio snippets to recognize people they had previously come into contact with. While the exact percentage of recognition accuracy was not the major point of focus in this qualitative probe, the researchers did stress that both the better image quality and the use of a context-aware filtering are important when it comes to reducing errors in the system in real-world scenarios.

Lin et al. [14] proposed a guiding system in which visually impaired people can navigate and avoid any obstacles using smartphone. The system is a combination of the mobile application with backend server and provides two modes to guarantee the functionality whatever the network availability. By taking advantage of Faster (R-CNN algorithm or YOLO algorithms) the server side part detects different types of obstacles and the smartphone performs face and stair recognition locally. As well, the system also calculates the approximate distance and direction of detected objects, thus, the users get spatial understanding. The information that is recognized is communicated with the help of voice notifications to the user and they are generated by a text-to-speech engine. Experimental results showed a recognition accuracy of 60% which was considered to be sufficient for the environmental identification in real-time. This approach makes it possible to successfully navigate to complex environments without sensors that have already been placed.

Král et al. [15] have proposed a low cost and multifunctional wearable equipment in the form of glasses to increase the independence of visually impaired persons. The system has been made up of a Raspberry Pi Zero 2W and an Arducam IMX519 camera to bring together real-time text recognition, AI-based scene description and remote volunteer help. Users communicate via a tactile user interface and have an audio feedback to provide situational awareness to the user. In performance evaluations, the OCR module had an error rate of 9% in the case of standard fonts, but accuracy was reduced quite significantly when it had to do the processing of artistic text styles. While this device was much touted due to its light design and full functionality, from the results it's apparent that more optimization in processing and hardware miniaturization needs to be done.

Ronald et al. [16] proposed a real-time object detection system to help visually impaired people, basing on a deep learning and computer vision. The system makes use of some frameworks such as (YOLO, TensorFlow, COCO-SSD and OpenCV) to detect common objects as they happen in daily life through the use of a camera. The objects that are detected are classified and converted to audio feedback to help the users to understand the surrounding. Experimental results showed good performance with the confidence values ranged from 0.87 to 0.92 for different objects category. The study shows that proposed approach is appropriate for real-time assistive applications and increases the user independence and awareness of environment.

Marwa Obayya et al. [17] presents the intelligent indoor object detection based assistive framework for the blind people using the optimized deep learning architecture. The system makes use of gaussian filtering to eliminate noise, (YOLOv12) for object detections in real-time, (DenseNet16) for deep feature extraction and BiGRU with attention mechanism for accurate object classification. Furthermore, Ivy Optimization Algorithm is used in hyperparameter tuning which gives a better overall performance of the system. The framework was tested on an indoor object detection data set with 10 object categories and a better recognition accuracy of 99.74% was obtained. The results showed the proposed method is more robust, have better inference time and adaptability to dynamic indoor environments than the existing methods. This study confirms the success of the combination of advanced detection, feature learning and optimization algorithms for reliable navigation assistance to the visually impaired users.

Jin et al. [18] proposed the concept of a smart cane-based face recognition system for visually impaired people for visual recognition of people in their surrounding. The idea behind this system is that a camera placed in the glasses is used to take real time pictures and transmit them to a mobile computer where they are fed to the system to perform the face detection in the pictures using AdaBoost and feature extraction methods of Modified Census Transform (MCT). Face classification is achieved with compressed sensing and using L2-norm so as to reduce the calculation complexity but to ensure the robustness. The recognized identity is fed back to the user by utilizing different vibration patterns from a microcontroller that is incorporated in the cane without the requirement of an audio feedback. Experimental evaluation on 10 known individuals and 6 participants showed recognition success rate of 93.33% which is reliable real time performance. The study emphasizes the effectiveness of tactile feedback along with computer sight in the aid of social interactions in the blind users.

The review of the literature and comparison presented in Gap Analysis Table 3 suggest that there are some critical problems that are yet to be addressed in relation to assistive technology to the visually impaired. Although current systems show a high level of accuracy in controlled or laboratory-based systems with accuracies of over 97 percent in certain tasks such as face or object recognition the perceived accuracy and reliability of the current systems tend to fail in unpredictable and real-world conditions. There is a common trade-off between computational efficiency and portability; systems based on small model deep learning can perform better at the device, but can be less context rich as compared to large language models (LLMs) which are more expensive to run in online scenarios. Besides, most existing prototypes are limited by environmental ethics e.g. severely limited functionality in lowest light situation or being dependent on certain local infrastructure e.g. iBeacons. This trust gap has also been established in the user experience studies as people like to use such high tech systems as an addition to the traditional aids such as the white cane instead of a solution by itself. This study attempts to fill these gaps by looking at a more intelligent, low latency and environmentally sound system with more emphasis on the autonomy of the user without compromising the safety or usability of the wearable.

Proposed Methodology

This research is carried out by experimental and applied research methodology which is concerned with the design and development, implementation and evaluation of an AI based mobile assistive system for visually impaired people. The aim is to experimentally demonstrate the feasibility, accuracy and performance of applications of real-time object detection, facial recognition and voice-guided navigation based on on-device artificial intelligence.

Data Sourcing Methodology

The data utilized by the VisionMate system is from two primary sources, in order to support the object detection/ recognition capabilities in real time and the facial recognition functionalities. These data sources are carefully selected to provide the efficiency, accuracy and privacy of the users.

Open-Source Datasets

For the object detection tasks a modified model of YOLOv8-Nano that has already been trained on a huge data of common objects like vehicles, furniture, household objects and pedestrians. Leveraging this worldly knowledge makes it possible for the model to recognize a wide variety of objects with a decrease in the complexity of training, the computing requirements, and the development time. This way, the accuracy of detection is quite high and real-time performance on the mobile devices are possible.

User-Generated Data for Facial Recognition

For facial recognition, image data is directly acquired from the registered guardian via mobile application interface. These images are processed locally on the device to create high-dimensional numerical feature embeddings using the MobileFaceNet model.To ensure that the user's privacy and data security is dependent on the privacy and data security of the devices used to capture and transmit the facial images.To ensure that the user's privacy and data security, raw facial images are not permanently stored. Instead of storing the numerical feature embeddings for unique features of the face, only securely stored numerical feature embeddings are stored in the local database. This approach is guaranteed to not misuse the personal data, while still have a reliable working performance in recognition.

Parameters

The main data parameters derived from the processing process are bounding box location (x, y, width and height co-ordinates), object class labels (e.g. person, car, chair) detection confidence scores, and facial features embedding of dimension 192. These parameters taken collectively helps to provide the correct localization, classification and recognition of an object.

Data Processing Workflow

The whole process of data processing is broken into four consecutive steps:

Data Acquisition:

The smartphone camera is used to capture the live video frames and the guardian contains the application interfaces to upload the faces images.

Preprocessing:

Frames that are captured are resized to 320 x 320, and normalized according to the input specifications of the YOLOv8-Nano model so that they will have the best computational efficiency.

Inference:

YOLOv8-Nano and MobileFaceNet are deployed locally in the device GPU/CPU respectively to detect the objects and extract the facial features in real-time.

Data Utilization:

The classes of the objects identified and identities are translated into audible feedback through Text-to-Speech (TTS) engine to give instant feedback to the visually impaired users.

System Workflow

Figure 3.1: Proposed System

The proposed system is planned to be operated primarily by using the on-device processing for the purpose of guaranteeing real-time performance, preserving the privacy and reducing the reliance of the internet connectivity. The mobile application is made with the help of the Flutter framework and this is the main control unit which integrates the smartphone camera, GPS, speaker, and processing with the help of embedded Artificial Intelligence models. work flow of the system starts with the continuous capture of real-time visual data using the smartphone camera. The gained frames are preprocessed and fed to light-weighted AI models which are implemented with TensorFlow Lite(TFLite) framework. The object detection module is made up of YOLOv8-Nano which determines the surrounding objects while the facial recognition module is made up of MobileFaceNet for facial recognition of the registered people. Meaningful audio feedback from processed results is achieved using a Text-to-Speech (TTS) engine to inform the user.2 For navigation and security purposes, the system has location tracking (GPS engine) and voice-guided navigation. In addition to this, a secure Guardian Mode is implemented with the help of the Firebase Realtime Database that proposes authorized guardian to track the real-time location of the user. This combination of offline and online architecture is capable of ensuring fast response, safe handling of data and reliable 24x7 assistance for the visually impaired.

Models Training & Implementation Methodology

The proposed system is implemented on light weight deep learning models and these models are optimized to run in real time at device. The selected models have high detection accuracy, low detection latency and can make good use of their resources hence can be run in regular smartphones.

YOLOv8-Nano 

YOLOv8-Nano is used to detect the objects in real-time as it has a small structure and can run the inference in the quickest manner. The model is taking continuous video frames which are captured by the smartphone camera and it is detecting different categories of objects by generating bounding boxes and marking them with corresponding classes. Its lightweight framework makes it capable to perform the operations in the device effectively and its dependable detection functionality that is important in providing real-time environmental awareness for the visually impaired users.

MobileFaceNet

MobileFaceNet is applied to facial recognition is efficient in terms of real time face detection and extraction of feature with low computational cost. The regions in the face are detected by the model and extract discriminative feature embeddings and store them safely in the local database at the registration stage. The extracted embeddings are then compared with the stored templates at the runtime so to identify registered persons so as to establish a reliable recognition and user privacy is still preserved.

Voice Navigation and Feedback

The system has the fact of route guidance in real time, which is based on navigation services GPS. The deployment of a Speech-to-Text module enables the voice recognition of inputs to enter one's destinations using the help of voice. The audible responses are the navigation instructions, objects detected and results of recognition which are converted into text to speech (TTS) engine and enable hands-free and intuitive interaction.

Primary: Visually Impaired User

Description:

Processes camera frames using an offline YOLOv8-nano quantized (int8) model to detect and localize objects in real-time, returning filtered bounding boxes.

Trigger:

The system background loop passes a decoded image frame to the ObjectDetector service.

Preconditions:

1. Camera permissions are granted.

2. The yolov8n_int8.tflite model and labels.txt are successfully loaded into the TFLite interpreter.

Post conditions:

Success: Returns a filtered list of Detection objects containing labels, confidence scores, and bounding box coordinates.

Normal Flow:

1. System receives a captured image frame.

2. System resizes the image to the required 320x320 input size and normalizes RGB pixel values to 0.0-1.0 (NHWC format).

3. System runs the TensorFlow Lite interpreter on the preprocessed image.

4. System parses the raw tensor output array into bounding box coordinates and class probabilities.

5. System filters out any predictions below the 0.3 confidence threshold.

6. System applies Non-Maximum Suppression (NMS) using an Intersection over Union (IoU) threshold of 0.5 to remove overlapping duplicate boxes.

7. System returns the final refined list of detections to the calling screen.

Alternative Flows:

1. A1: No Objects Detected: If no predictions pass the 0.3 confidence threshold, the system returns an empty list.

Exceptions:

1. Model Uninitialized: If detection is called before the TFLite model loads, the system throws a "Model not loaded!" exception.

2. Inference Failure: If the TFLite interpreter crashes during execution, the system catches the error, prints the debug log, and safely returns an empty detection list.

 None

When using object detection.

Special Requirements:

Low latency.

Offline capability.

The DFD Level 0 of VisionMate system is shown in the Figure 5.1. The inputs are sent to the system by the visually impaired user in the form of camera feeds, voice commands, navigation requests and location sharing and the responses received in the form of audio feedback and voice navigation guidance. The system receives management information and known face registration information in the guardian side and provides face registration confirmation information and live location information for assisting in safety management. On the whole, this diagram provides a high level impression of the way the data gets in and out of the system, and provides the essential interactions without detailed processing information.

Figure 5.2: DFD level 1Figure 5.2: DFD level 1Data Flow Diagram Level 1

Figure 5.2: DFD level 1

Figure 5.2: DFD level 1

The DFD Level 1 of VisionMate is represented in the Figure 5.2 in which the system is partitioned into major internal processes and data stores. The Visually Impaired User controls the interaction of the system by providing the system with camera feed, navigation requests and voice commands to the system. The feed from camera is processed by an Object Detection Processing to detect the objects around and sends the result back in form of audio feedback. The same camera feed is also used by the Face Recognition Processing which searches the Face Database to see if a match is found and then give the audio feedback as to who the person is. For safety support the user is able to send location sharing request to Guardian Mode Processing, this stores the updates of location data and communicate to User Account Data in case there is a need to changing the user account. The Guardian has access to this module which will supply the live location information of the user.

Data Flow Diagram Level 2

Figure 5.3: DFD level 2 Navigation

The following figure 5.3 is the DFD Level 2 of the feature of Navigation & Voice Command in VisionMate. First it is the Visually Impaired User who speaks a command which is processed by the Receive Voice Input to process them into destination text. The system can also make use of places, which had previously been saved in the Favourite Locations data store. The process of Determine Destination select final destination either on the basis of spoken input or saved locations and send destination request to Get Route. The Get Route process uses the Maps Service (External) to Get Route information by sending the Maps Service a route query and receiving route information. After the route is generated, the route steps are then passed into Provide Voice Guidance which translates the steps of the navigation into spoken instructions and returns voice navigation guidance as spoken instructions to the user.

Figure 5.4: DFD level 2 object detectionFigure 5.4: DFD level 2 object detection

Figure 5.4: DFD level 2 object detection

Figure 5.4: DFD level 2 object detection

In VisionMate, Figure 5.4 is the DFD Level 2 for Object Detection feature. The camera feed is given by the Visually Impaired User and the system firstly captures a camera frame to get an image frame. This image frame is then passed to the Run Object Detection Model process which uses the stored Object Detection Model (model parameters) to detect objects in the scene. The output of the model is a list of the detected objects with their labels and confidence values and is passed to Interpret Detection Results to get a nice summary of the objects. Finally, the system transmits this summary information to Provide Audio Feedback which changes this information into speech and audio outputs the feedback to the user and he can understand his surroundings without seeing the screen.

Figure 5.5: DFD level 2 Face Recognition

This level 2 DFD figure 5.5 shows the Face Recognition in VisionMate. Visually Impaired User from the feed from the camera and the system captures a frame from the camera feed. The system then identifies the face and removes it off in the image frame so it is only the face in the frame. Then, it removes face features (embedding) on the face picture. This is what is termed as face embedding and it is forwarded to the Match Face with Database process where it is matched to the existing face embeddings in Face Database (Registered Faces). When a result of a match is found (recognized person or unknown) then the system provides an audio feedback and informs the result to the user hence a user can identify people without visual interaction.

System Design

System Architecture Diagram

Figure 6.1: System ArchitectureFigure 6.1: System Architecture

Figure 6.1: System Architecture

Figure 6.1: System Architecture

This System Architecture Diagram demonstrates how the VisionMate mobile app is functioning, and how different items are interacting with each other. The Guardian and Visually Impaired User interfaces with the app with their own user interfaces written with the Flutter. The primary app logic is for stuff such as navigating and voice/button controls and guardian mode. For smart help, the app uses AI models such as YOLOv8 Nano for object detection and mobile faceNet for face recognition which is mounted on for smart help in the phone, so that the detection may work swiftly on the phone. Registered face embeddings are stored locally in SQLite in order to match faster. The system is connected with the Firebase Authentication (For login) and the Firebase Realtime Database (To send and receive user's location on real time basis, so that guardian can monitor location if required). Finally, the app makes use of phone hardware such as the camera, GPS, microphone and speaker to capture live input as well as giving the visually impaired user audio feedback.

Class Diagram

Figure 6.2: Class DiagramFigure 6.2: Class Diagram

Figure 6.2: Class Diagram

Figure 6.2: Class Diagram

The Figure 6.2 has the Class Diagram of the VisionMate system containing the main classes and how they are related to each other. The User class is used to store common information such as userId, name, email/phone, password and role and it is extended by two specialized classes; Guardian and VisuallyImpairedUser. The Guardian has the charge of safety and support functions - live viewing of the location and face registration. Face data is stored in the form of FaceProfile and FaceEmbedding where one profile has one embedding for face recognition. The VisuallyImpairedUser class is associated with important assistive features and other related data, for example, the LocationData which is used for GPS tracking of the user, FavouriteLocation which is used to store important locations for the user, and DetectionResult which is used to store the output of detecting objects. Several DetectedObject entires with labels and confidence values can be contained in each detection result. For the purpose of a navigation, the user creates a NavigationRequest, which could generate a Route, and the route is divided into multiple RouteSteps for step-by-step guidance of the user. Finally, the connecting of the guardian to the visually impaired user is ensured by GuardianLink to ensure that the monitoring and support features that are only available when accounts are connected.

This Figure 6.3 is the Sequence Diagram of VisionMate which describes the step by step interaction between two actors (Visually Impaired User and Guardian) and system components. Firstly, the Visually Impaired User logs in to the VisionMate app and VisionMate app sends the credentials for validation to Auth Service. After successful login the app also checks from User/Link Database to know user have associated with a guardian or not. Next, the user opens the Object Detection where the app will send frames from the camera into the Camera + AI Model and receives the objects detected in the frame. These results are then translated into speech and TTS (Audio Feedback) and voiced out as audio output. For the navigation, the user triggers the voice command from the shortcut of the volume up, and then tells the destination and the app asks the GPS / Maps Service for the navigation of the route. The returned instructions for navigation are again provided to the user by means of the audio feedback. Finally, the Guardian may ask to View Live Location and the app will get the user's coordinates from the database and display live location to the guardian completing the safety monitoring flow.

Implementation details

VisionMate application has been built using Flutter and Dart as a platform in order to enable the cross platform development. The basic components were developed as stand-alone modules like authentication and account connection; object recognition, face recognition, navigation and guardian live location tracking. The stream of the camera is also processed in real time and the results of detection and recognition generated by the lightweight on-device AI models. Local storage of registered face embeddings can be used to match them fast, whereas logging in and live location sharing is provided by the services from Firebase. All user products are delivered in the form of text-to-speech so that it can also be accessed by blind people.

Constraints

VisionMate will also be susceptible to several restrictions, effects the way system built, and how the system works in real life scenario. The application must also be supported by normal mobile devices and hence, the high quality and rate of processing, the quality of the camera and battery life limit the precision and real-time performance of the on-determination AI models. The system also needs to have user permissions to access the cameras, microphones and location which will not be possible without the user permissions. Navigation and live location sharing is based on GPS and in other situations, on the use of the internet and third-part applications such as Firebase and Maps APIs and weak signals or interference with the internet connection can result in lower reliability. In addition, the face registration quality and face registration under face light condition will determine the face recognition output process. The design had these limitations in consideration ensuring that the system is not a heavy load and easily accessible and convenient in portable gadgets.

Assumptions

Device Requirements: Access users all have a smartphone with a working camera (front/rear) and GPS receiver.

User Competency: Guardians are able to accurately register face profiles in initial setup process without occlusion, poor lighting.

Data Accuracy: GPS coordinates are of reasonable quality in the open (users are in expected geographic regions)

Operational Training: Users are adequately trained to use the mobile navigation system.

Network Availability: Guardians have intermittent synchronization connectivity with the fire base. 

System Constraints

Restrictions

Only smartphone based system (no use of additional hardware: smart cane, glasses, sensors)

The voice commands and the audio feedback are the things that support the English in the application right now.

The face registration and face management is limited to the Guardian account only.

Live location tracking is available only to a linked Guardian (linking is required).

Features are dependent on permissions given to users and connectivity (camera/mic/location permissions; navigation may require the internet for routing).

Limitations

AI accuracy can be affected by low lighting, rain/fog, crowd, fast camera movement and can affect object detection and face recognition accuracy.

Continuous using camera + AI processing could lead to excessive battery consumption, also sometimes heating of the device in case of long sessions.

No matter what the software, no matter how good the camera quality is or how much RAM or processor the phone has, the quality of the system performance is dependent on the hardware of the phone, and so it can be possible that with older or low-end phones that the quality of the output can be slower or less accurate.

This section presents decision tables for the critical functional modules of the VisionMate system. A decision table is a structured representation that captures all possible combinations of input conditions and their corresponding system actions, ensuring that every logical branch within a module has been considered and tested. For each key feature including object detection, face recognition, voice command handling, fall detection and SOS escalation the conditions that influence system behaviour are identified alongside the actions the system must take for each combination. This approach complements the extended test cases by verifying that edge cases and boundary conditions are explicitly accounted for, and provides a clear traceability link between system logic and the test scenarios defined in the previous section.

    var input = Float32List(1 * inputSize * inputSize * 3);

    final formattedInput = input.reshape([1, inputSize, inputSize, 3]);

    final output = Float32List(1 * 192).reshape([1, 192]);

Fall Detection and Safety Prompt

void _onAccelerometerData(AccelerometerEvent event) {

    if (_isFallPromptActive) return;

    final now = DateTime.now();

    final magnitude = sqrt(

        (event.x * event.x) + (event.y * event.y) + (event.z * event.z),

    );

    if (magnitude < _freeFallThreshold) {

        _lastFreeFallAt = now;

        return;

    }

    final hadRecentFreeFall =

            _lastFreeFallAt != null &&

            now.difference(_lastFreeFallAt!) <= const Duration(milliseconds: 1200);

    final isImpact = magnitude > _impactThreshold;

    final cooldownPassed =

            _lastFallTriggerAt == null ||

            now.difference(_lastFallTriggerAt!) > const Duration(seconds: 20);

    if (hadRecentFreeFall && isImpact && cooldownPassed) {

        _lastFallTriggerAt = now;

        _triggerFallPrompt();

    }

}

This snippet detects a free-fall followed by impact and starts the safety prompt with countdown handling.

9. Future Work

• Multilingual support: extend TTS and voice-command grammars to support major local languages and dialects, and provide a localisation pipeline for prompts and help text.

• On-device fine-tuning and calibration: provide lightweight per-device calibration tools and optional on-device fine-tuning to adapt recognition models and thresholds to individual users and hardware variations.

• Emergency integration: implement configurable emergency endpoints (e.g., 1122) with consent-managed automatic SOS calls and hospital escalation paths, including safety confirmations and rate-limiting to avoid false alarms.

• Extended trials and validation: conduct structured field studies with visually impaired participants to evaluate usability, reliability, and social acceptability across environments.

• Privacy-preserving telemetry: build tooling to collect aggregate, anonymised metrics to improve models without leaking personal data.

• Deployment tooling: create device management and remote-update mechanisms for models and thresholds, and define monitoring/alerting for production deployments.

CONCLUSION

This paper presents VisionMate, an integrated smartphone assistive system that combines fall detection, offline face recognition, voice-guided navigation, object detection with audio feedback, and guardian-aware emergency escalation. The implementation focuses on delivering practical, privacy-preserving assistance for visually impaired users by keeping computation on-device where possible and providing reliable fallback paths when network services are unavailable.

Implementation evidence is provided by the artifacts generated and inserted into this document: the Decision Coverage Table (expanded to 12 rules) captures the principal runtime branches and escalation paths, while the Traceability Matrices (7.3.1, 7.3.3 and 7.3.4) map requirements to use cases, prototypes, and test cases. Key runtime thresholds and heuristics used in the implementation include the fall detection thresholds (free-fall and impact thresholds), a face-similarity cutoff of approximately 0.80 for offline recognition, and object-detection confidence / NMS settings (confidence ≈ 0.3, IoU threshold ≈ 0.5). These parameters are reflected in the decision rules and in the test cases referenced in section 7.3.

From an engineering perspective, the project demonstrates several practical contributions: a lightweight on-device recognition pipeline that reduces privacy and latency concerns, a decision-driven escalation flow that prioritizes user confirmation before emergency actions, and a fault-tolerant OTP / notification strategy with a documented fallback path. The inserted Results section (8.1–8.3) summarizes completion and correctness by referencing the traceability matrices and the implemented test cases, indicating that each documented use case has at least one corresponding test and prototype mapping.

Limitations remain and are acknowledged: the validation performed here is functional and demonstrative rather than a large-scale user study; sensor behaviour varies between device models and environmental conditions; and speech/voice interactions require additional localization and accessibility refinements for non-English users. These limitations are noted so that follow-up work can focus on robustness and generalization.

Future work therefore includes structured user trials with visually impaired participants, adaptive threshold tuning (device- and user-specific calibration), expanded multilingual support for voice feedback and prompts, and improved model-selection strategies to balance accuracy, latency, and battery consumption across a range of target devices. Together, these steps will strengthen the empirical claims suggested by the traceability and decision-coverage artifacts inserted into this document.

Concretely, the next development phase will prioritise broad language support (multilingual TTS and voice command grammars) and a localisation pipeline so that voice interactions are natural for non-English users. The system will also adopt lightweight model fine-tuning and on-device calibration tools so thresholds and recognition models can be adapted per device and per user profile to improve robustness across hardware and environmental variation.

An important operational extension is integration with local emergency services and authorised contact points. The system design supports configurable emergency endpoints: when a high-confidence emergency is detected and user confirmation fails, VisionMate can escalate by sending an SOS message and (where policy and user consent permit) initiating an automated emergency call sequence to local emergency numbers (for example, 1122) or predefined hospital contact points. Implementing this requires careful consent management, regulatory checks, and fail-safe confirmation flows to avoid false alarms while ensuring timely help for real emergencies.

Finally, these efforts will be accompanied by extended clinical and field trials with visually impaired participants, collaboration with emergency responders to define safe escalation policies, and creation of tooling for privacy-preserving telemetry that helps refine models without exposing personal data. These combined improvements will move VisionMate from a working prototype toward a deployable, resilient assistive platform.
