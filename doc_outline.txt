  Summary: Abstract | This document describes VisionMate, a smartphone-centred assistive system for people with visual impairments that combines on-device obstacle perception, offline face recognition, turn-by-turn voice navigation, and guardian safety synchronization in a single mobile application. The system implements a dual-zone safety model: a frontal zone for near-term collision detection and a lower path zone that estimates walkability and announces transitions between blocked and clear states. Edge inference is performed locally using YOLOv8n-int8 for object detection and MobileFaceNet for face embedding extraction (TensorFlow Lite), minimising cloud dependence for privacy-sensitive perception tasks. Interaction is audio-first, using text-to-speech for continuous situational feedback and speech input for destination entry; hardware volume-button triggers and inertial fall-detection enable quick user actions. A guardian mode shares realtime location and device status via Firebase (authentication and sync), while a local SQLite cache supports offline face matching. The implementation demonstrates that commodity smartphones can deliver an integrated, low-cost assistive solution that improves autonomy and safety for visually impaired users. | Contents

  Summary: Introduction | Visual impairment can severely limit an individual’s independence, safety, and confidence during routine activities. Tasks that sighted people take for granted—avoiding obstacles, recognising known people, and following a safe path—become difficult to perform unaided. Traditional aids such as the white cane help with immediate physical barriers but do not provide semantic scene information or person recognition. Modern assistive devices that offer richer functionality are often costly or require specialised hardware. | VisionMate is a mobile application that transforms an ordinary smartphone into an assistive companion for people with visual impairments. The app provides real-time object detection and tracking, offline face recognition for identifying registered contacts, voice-guided navigation, and a guardian mode for live location sharing. Core feedback is delivered via audio so the interface remains accessible without visual dependence. By combining on-device perception, offline caching, and optional cloud sync, VisionMate aims to deliver an affordable, holistic solution that enhances mobility, social interaction, and personal safety.

[1] proposed Accessibility Bot, a research prototype for assisting individuals with visual impairments (VIPs) to identify their friends and perceive facial information in the process of social activities. Integrated into Facebook Messenger, the system uses the phone's camera to identify faces in real time, and takes advantage of cutting-edge computer vision models that have been trained by using existing tagged photos on Facebook, that eliminates the need for manually creating training datasets. The application extracts important information about the faces, such as which face, location of the face in relation to the others, expressions and attributes like facial hair, that is fed back to the user via the audio feedback via a screen reader. While the underlying recognition algorithm was able to achieve greater than 97% accuracy on the PIPA dataset, in a study involving six VIPs over a period of seven days, it was noted that in practice "perceived accuracy" was often less because of issues with aiming the camera and inconsistent photo quality. Ultimately, the research was able to offer some deep insights into the practical problems of implementing assistive face recognition technology in uncontrolled environments outside of the lab.

- [L951] System Workflow (content lines: 2)
  Summary: Figure 3.1: Proposed System | The proposed system is planned to be operated primarily by using the on-device processing for the purpose of guaranteeing real-time performance, preserving the privacy and reducing the reliance of the internet connectivity. The mobile application is made with the help of the Flutter framework and this is the main control unit which integrates the smartphone camera, GPS, speaker, and processing with the help of embedded Artificial Intelligence models. work flow of the system starts with the continuous capture of real-time visual data using the smartphone camera. The gained frames are preprocessed and fed to light-weighted AI models which are implemented with TensorFlow Lite(TFLite) framework. The object detection module is made up of YOLOv8-Nano which determines the surrounding objects while the facial recognition module is made up of MobileFaceNet for facial recognition of the registered people. Meaningful audio feedback from processed results is achieved using a Text-to-Speech (TTS) engine to inform the user.2 For navigation and security purposes, the system has location tracking (GPS engine) and voice-guided navigation. In addition to this, a secure Guardian Mode is implemented with the help of the Firebase Realtime Database that proposes authorized guardian to track the real-time location of the user. This combination of offline and online architecture is capable of ensuring fast response, safe handling of data and reliable 24x7 assistance for the visually impaired.
